# Database Schema

---

# Database Overview

The Student Semester Registration System uses PostgreSQL as its primary database and Drizzle ORM as the database access layer.

Version 1 intentionally uses a simple schema to minimize complexity while supporting all required functionality.

The database is designed to support:

* Student authentication
* Administrator authentication
* Semester registration
* Attendance tracking
* Dashboard analytics
* Student management

---

# Database Design Principles

## Single Source of Truth

All student registration data is stored in PostgreSQL.

No duplicated records exist outside the database.

---

## Normalized Structure

Data should not be duplicated unnecessarily.

Each entity owns its own information.

---

## Future Expandability

The schema should support future features including:

* Multiple semesters
* Attendance records
* Course registration
* Academic departments

without requiring major redesign.

# Drizzle Schema

## users.ts

```ts
import {
  pgTable,
  uuid,
  varchar,
  integer,
  date,
  timestamp,
  text,
} from "drizzle-orm/pg-core";

export const users = pgTable("users", {
  id: uuid("id")
    .defaultRandom()
    .primaryKey(),

  name: varchar("name", {
    length: 255,
  }).notNull(),

  email: varchar("email", {
    length: 255,
  })
    .unique()
    .notNull(),

  passwordHash: text("password_hash")
    .notNull(),

  dob: date("dob")
    .notNull(),

  phone: varchar("phone", {
    length: 20,
  }).notNull(),

  attendance: integer("attendance")
    .notNull(),

  attendanceStatus: varchar(
    "attendance_status",
    {
      length: 20,
    }
  ).notNull(),

  semester: varchar("semester", {
    length: 50,
  }).notNull(),

  role: varchar("role", {
    length: 20,
  })
    .default("STUDENT")
    .notNull(),

  registeredAt: timestamp(
    "registered_at"
  ).defaultNow(),

  createdAt: timestamp(
    "created_at"
  ).defaultNow(),

  updatedAt: timestamp(
    "updated_at"
  ).defaultNow(),
});
```

# Dashboard Query Requirements

The following analytics depend on this schema.

---

## Registrations Today

```sql
SELECT COUNT(*)
FROM users
WHERE role='STUDENT'
AND DATE(registered_at)=CURRENT_DATE;
```

---

## Total Students

```sql
SELECT COUNT(*)
FROM users
WHERE role='STUDENT';
```

---

## Low Attendance Students

```sql
SELECT COUNT(*)
FROM users
WHERE attendance < 75;
```

---

## Average Attendance

```sql
SELECT AVG(attendance)
FROM users
WHERE role='STUDENT';
```

# Database Constraints

Mandatory Rules

1. Email must be unique.
2. Passwords must always be hashed.
3. Attendance must remain between 0 and 100.
4. Role must be STUDENT or ADMIN.
5. Registration timestamps are system generated.
6. Database access must occur only through Drizzle ORM.
7. Direct SQL queries are prohibited outside migrations.
8. All writes must pass validation before persistence.

This schema serves as the single source of truth for the Student Semester Registration System database.
